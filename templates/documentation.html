Schedule Generation Settings

These settings allow the user to change the way that the schedules are generated. Please be aware that changing these settings will effect all users.


Chairs per RN The maximum number of patients that each RN can be assigned at a time.
Open Time The time that the chemotherapy facility opens. Restricts the time that RNs can be start, and the time that appointments can be scheduled
Close Time The time that the chemotherapy facility closes. Restricts the time that RNs can work until, and the time that appointments can be scheduled
Start Delay The amount of minutes after the open time before any appointments can be scheduled

The Admin Page

This is a page by page explanation of the admin site


Login Page: To log into the admin page the user must be a designated Superuser, the process for which is explained in section XXX. Once you have obtained your credentials, you can simply log into the main website with them and append “/admin/” to the main page to gain access to the admin page.
The Main Page: In the top right of the main page, you should see some text saying WELCOME, USERNAME followed by three options: ViewSite, which will take you back to he main page of the scheduling application, Change Password which will take you to the password change page, and Log Out which will log the user out and bring the user to the logout page. On the left side of the page under the AUTHENTICATION AND AUTHORIZATION bar there are two rows, one for users and one for groups (a group of users with a set of permissions or traits). By clicking the title “Users” or “Groups”, the admin site will redirect you to the appropriate pages to edit the current users or groups in the database. Clicking the pencil icon to the far right of the row will have the same effect. By clicking on the + button on either of these rows, the admin site will redirect you to the “add” page for users or groups.Below this there is the PATIENTSCHEDULING section which works in a very similar fashion to the section above it. The main differences are that the rows reference to the chemotherapy drugs, nurse schedules in the system, and Saved Time Slots Inputs, which reference the sets of time slots that you can save and load in the main application. One thing to note here is that the last two rows in this section can only be edited on the admin page as you can only add a new nurse schedule or set of time slots in the main application Finally, to the left of these two sections is the Recent actions tab, which record the recent actions (such as editing a schedule) that all users have taken on the admin site. One thing to note here is that it does not record the action of logging in or accessing the site.
Groups Select Page: Use the search bar to find the group you wish to edit. By clicking the check box to the right of the name, you can then use the actions drop down menu to delete the ones you have selected. By clicking on a group’s name you get redirected to the change group page. To get to the group add page click the small add button on the top left of the page under the change password link.
Groups Add Page: To start, type the name of the group you wish to create in the NAME textbox. Then, either use the scroll bar, or the search box to select the combination of permission for the the members of the group. After clicking on the set of permissions, click the right arrow in between the two boxes to move that set of permissions into the chosen permissions box. This can be reversed by clicking the item in the chosen permissions box and proceeding to click the left arrow. If you feel that it is convenient, you can click the choose all to add all permissions or the remove all to remove all permissions from the group. At the bottom you can click the corresponding buttons to save and make a new group, save your progress and continue editing, or save and exit back to the main page (the save button).
Change Group Page: This page is identical to the add group page except the user can also click the red delete button at the bottom left to delete the group.
Users Select Page: Use the search bar to find the user you wish to edit. Conversely you can also use the several filters provided to you under the FILTER section to the right. By clicking the check box to the right of their name, you can then use the actions drop down menu to delete the ones you have selected. By clicking on a user’s name you get redirected to the change user page. To get to the user add page click the small add button on the top left of the page under the change password link.
Change User Page: The first textbox is used to change the username, under the personal info tab the user can fill out the first name, last name, and email address if he or she so desires. Under the permissions tab there are three check boxes which are active, staff status, and superuser status. Their effects are described on the page. Under these you can select which groups you wish the user to belong to and click the right arrow to move them to the set of chosen groups. Below this the user will see set of 2 boxes for user permissions (see description of the add group page for further detail of how to use these boxes). Finally, the last section is if important dates such as last login and date joined, which can also be changed. At the bottom you can click the corresponding buttons to delete the user, save and make a new user, save your progress and continue editing, or save and exit back to the main page (the save button).
Users Add Page: To start, type the name of the user you wish to create in the NAME textbox. Then, enter the password for the new user. After this is filled out, please retype your password into the password confirmation box to ensure you typed the correct password. At the bottom you can click the corresponding buttons to save and make a new user, save your progress and continue editing, or save and exit back to the main page (the save button).
Chemotherapy Drug Select Page: This page will display all the drugs in the database. By clicking the check box to the right of their name, you can then use the actions drop down menu to delete the ones you have selected. By clicking on one of the drug name you get redirected to the change chemotherapy drug page. To get to the chemotherapy drug add page click the small add button on the top left of the page under the change password link.
Chemotherapy Drug Add Page: To start, type the name of the drug you wish to create in the NAME textbox. Then, enter the earliest and latest times the drug can be scheduled for any given day (you can click the now button to automatically enter the current time). After this is filled out, please enter any additional rules or pertinent information about the drug into the “Other Rules” box to be displayed when that drug is selected at the calendar view page. At the bottom you can click the corresponding buttons to save and make a new user, save your progress and continue editing, or save and exit back to the main page (the save button).
Change Chemotherapy Drug Page: This page is identical to the Chemotherapy Add Page except one can delete the drug by clicking the red delete button and the one can view the history of the drug relative to the database by clicking the grey history button at the top right under the log out link.
RN Schedule Select Page: This page will display all the RN schedules in the database. By clicking the check box to the right of their name, you can then use the actions drop down menu to delete the ones you have selected. By clicking on one of the RN schedule names you get redirected to the change RN schedule page.
Change RN Schedule Page: Fill out the name box to change the name of the saved schedule. Below this you can click the corresponding buttons to delete this schedule, save your progress and continue editing, or save and exit back to the main page (the save button).
Time Slot Inputs Select Page: This page will display all the sets of time slot inputs in the database. By clicking the check box to the right of their name, you can then use the actions drop down menu to delete the ones you have selected. By clicking on one of the time slot input names you get redirected to the change time slot inputs page.
Change Time Slot Inputs Page: Fill out the name box to change the name of the saved set of time slot inputs. Below this you can click the corresponding buttons to delete this set of time slot inputs, save your progress and continue editing, or save and exit back to the main page (the save button).
Logout Page: This page simply informs the user that he/she has been successfully logged out. By clicking the blue link under the text called log in again the user will be brought back to the login page for the application.
Change Password Page: As suggested by the name this page is for changing the user’s old password to a new one. Simply follow the instructions provided on this page by typing in the provided text boxes and click the CHANGE MY PASSWORD button on the middle right side of the page to change your password.

Technologies Explanation:

This section explains how the technologies fit together.


Glossary of terms

Python: The programming language we plan on using in the back-end of the code
Django: A high level open-source, Python, web application framework which follows MVC
CSS (Cascading Style Sheets): A language used to describe the look and format of HTML
Javascript A programming language used to create interactive effects in web applications
SQLite: The Django built in database
Postgressql: A standalone database that we are using for our hosted site/td>
Ajax: A means of communicating from client to server in order to retrieve information from the server.
Heroku: A way to deploy websites using a cloud server
Github: Version control software that allows for simultaneous editing
Bootstrap: An open-source, front-end framework that contains CSS stylesheets, and Javascript extensions

Overview:

 This web app was developed using the Pycharm IDE as it already had an excellent interface for building web tools using Django and Python. Python was used for the backend development which included the algorithm for generating a schedule, communicating with the database, customizing the admin site, and grabbing information from the textboxes on the actual website. For the font end we used the classic combination of HTML5 and CSS (enhanced with Bootstrap libraries) to design the web pages and make them aesthetically pleasing. We paired these with Javascript to add functionality to the various links and buttons that were placed on the website. Many of our Javascript functions utilized Ajax to call the various database related functions we had written in Python to either save, delete, or query for data (using Postgressql on Heroku, SQlite on local host). The website itself is hosted on Heroku, which is a cloud server and is a temporary place to use the website until the client is ready to move the web app to one of their own servers. Finally, we used git as our version control software so that all team members could work on the project simultaneously without fear of overwriting or corrupting each other's work.
