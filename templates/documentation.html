Preface

Vocabulary

Definitions of common terms used.


RN Schedules Location where users can add/remove RNs and add their work details.
Pod Team Team associated with RN. Used to identify who works with each other as a team.
Start Time The time where a RN officially starts work.
Lunch Time The time where a RN officially starts his/her lunch.
Lunch Duration The duration of RN's lunch break.
End Time The time where a RN officially ends work.
Reserved Time Periods Used to place mandatory appointments for a RN. Example: If Nurse 1 knows he/she has an appointment that can not be missed for 1 hour at 9 am. You would use section of the form to input it.
RN Number Unique ID associated with RN. Look at left most column at RN Schedules when using the new Schedule page to find the ID.
Chair Number Represents which chair you want to assign the mandatory appointment. Total number of chairs per nurse can be changed at the settings page.
Needed Timeslots Location where users can input the number of different appointments that users want. Example: 30 30 minute appointments, 15 1 hour appointments, etc
Prioritize Longer Appointments If checked, the algorithm will try to fit as many longest appointments and work its way down to smaller appointments. If not checked, priority will be given based on the order of the list from top to bottom.
Prepopulate Timeslots If clicked, it will autofill the timeslots selections with every single timeslot and autofill the total needed with 0.
Error Message An input given was incorrect. Please fix the error listed and try again.

Creating a New Schedule

The Calendar view

This page is where users can see the final schedule for the RNs. Each column is broken down by the hour and each row block represents a pod and their respective nurse.


Pod Each pod is represented by a solid grey filled bar. Right below are the respective RN for the pods. A pod can be collapsed to make it easier to view other pods.
Nurse / Chairs Each RN is represented by their ID and chair in the pattern #:A where # represents the ID and A represents their chair. They can be found in the very left column of the calendar. The rows next to the RNs represent their respective chairs.
Appointment The color bars shown represent all possible appointments. Each appointment length has their own unique color code. Each appointment can be clicked on to show more specific details such as start time, end time, and duration.
Striped Bars The blue striped bars represent the RN's lunch time. The grey striped bars represent times when they are not available due to early end time or later start time.
Dropdown Menu The dropdown menu represents the drugs that are available for the patients. Each drug has specific rules and any appointments that can not satisfy the rules are greyed out.

How to use: The calendar shows all the appointments that can fit for each nurse. The appointments that can not be fitted are shown in a red alert box at the top and means that the user has inputted too many appointments. RNs be collapsed by their respective pods for easier visibility. Each appointment is color coded by their length and be clicked on to show extra details. The drop down menu in the top left corner gives the user the ability to show which appointment can satisfy the selected drug's rules. The blue-striped bars represent RN's lunch time and the grey-striped bars represent when they are not available. Users also have the ability to save and print their schedule which can be found at the top right of the calendar.

Viewing Saved Schedules

Schedule Generation Settings

These settings allow the user to change the way that the schedules are generated. Please be aware that changing these settings will effect all users.


Chairs per RN The maximum number of patients that each RN can be assigned at a time.
Open Time The time that the chemotherapy facility opens. Restricts the time that RNs can be start, and the time that appointments can be scheduled
Close Time The time that the chemotherapy facility closes. Restricts the time that RNs can work until, and the time that appointments can be scheduled
Start Delay The amount of minutes after the open time before any appointments can be scheduled

The settings are saved to the text file called UserSettings in YAML format.
