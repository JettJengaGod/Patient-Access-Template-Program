Preface

Vocabulary

Definitions of commonly used terms.


RN Registered nurse.
Pod A single letter used to identify a team of nurses who can help one another with patients.
Appointment A scheduled time slot that has been assigned to a RN, at a particular chair, with a given start and end time.
Time Slot Some specified amount of time that will attempt to be scheduled as an appointment.

Navigation

Functionality

Creating a New Schedule

RN Schedule A list of nurse availability for a given day including their start time, lunch time, leave time, and the pod they are working within.
RN Number Unique ID associated with RN. Look at left most column at RN Schedules when using the new Schedule page to find the ID.
Pod Team A single letter used to identify a team of nurses who can help one another with patients.
Start Time The time when a RN officially starts work.
Lunch Time The time when a RN officially starts his/her lunch.
Lunch Duration The duration of RN's lunch break.
End Time The time where a RN officially ends work.
Reserved Time Periods Used to place mandatory appointments for a particular RN before generating the schedule. Example: If Nurse 1 knows he/she has an appointment that can not be missed for 1 hour at 9 am. You would use section of the form to input it.
Chair Number Represents which chair you want to assign the mandatory appointment. Total number of chairs per nurse can be changed at the settings page.
Needed Timeslots Location where users can input the number of different appointments that users want. Example: 30 30 minute appointments, 15 1 hour appointments, etc
Prioritize Longer Appointments If checked, the algorithm will try to fit as many longest appointments and work its way down to smaller appointments. If not checked, priority will be given based on the order of the list from top to bottom.
Prepopulate Timeslots If clicked, it will autofill the timeslots selections with every single timeslot and autofill the total needed with 0.

The Calendar view

This page is where users can see the final schedule for the RNs. Each column is broken down by the hour and each row block represents a pod and their respective nurse. Users also have the ability to save and print their schedule which can be found at the top right of the calendar.

Pod Each pod is represented by a solid grey filled bar. Right below are the respective RN for the pods. A pod can be collapsed to make it easier to view other pods.
Nurse / Chairs Each RN is represented by a unique ID and is responsible for some number of chais as specified in the settings. The rows next to the RNs represent their respective chairs.
Appointment The color bars shown represent all scheduled appointments. Each appointment length has their own unique color code. Each appointment can be clicked on to show more specific details such as start time, end time, and duration.
Striped Bars The blue striped bars represent the RN's lunch time. The grey striped bars represent times when they are not available due to early end time or later start time.
Dropdown Menu The drop down menu in the top left corner gives the user the ability to select a chemotherapy drug and only the appointments which can satisfy the selected drug's rules will be shown.

Viewing Saved Schedules

Schedule Generation Settings

These settings allow the user to change the way that the schedules are generated. Please be aware that changing these settings will effect all users.


Chairs per RN The maximum number of patients that each RN can be assigned at a time.
Open Time The time that the chemotherapy facility opens. Restricts the time that RNs can be start, and the time that appointments can be scheduled
Close Time The time that the chemotherapy facility closes. Restricts the time that RNs can work until, and the time that appointments can be scheduled
Start Delay The amount of minutes after the open time before any appointments can be scheduled

The Admin Page

The Main Page:

In the top right of the main page, you should see some text saying WELCOME, *USERNAME* followed by three options: "ViewSite", which will take you back to he main page of the scheduling application, "Change Password" which will take you to the password change page, and "Log Out" which will log the user out and bring the user to the logout page.

On the left side of the page under the AUTHENTICATION AND AUTHORIZATION bar there are two rows, one for users and one for groups (a group of users with a set of permissions or traits). By clicking the title “Users” or “Groups”, the admin site will redirect you to the appropriate pages to edit the current users or groups in the database. Clicking the pencil icon to the far right of the row will have the same effect. By clicking on the + button on either of these rows, the admin site will redirect you to the “add” page for users or groups.

Below this there is the PATIENTSCHEDULING section which works in a very similar fashion to the section above it. The main differences are that the rows reference to the chemotherapy drugs, nurse schedules, and Saved Time Slots Inputs, the last of which reference the sets of time slots that you can save and load in the main application. One thing to note here is that the last two rows in this section can only be edited on the admin page as you can only add a new nurse schedule or set of time slots in the main application.

Finally, to the left of these two sections is the Recent Actions tab, which record the recent actions (such as editing a schedule) that all users have taken on the admin site.
Groups Select Page: Use the search bar to find the group you wish to edit. By clicking the check box to the right of the name, you can then use the actions drop down menu to delete the ones you have selected. By clicking on a group’s name one gets redirected to the change group page. To get to the group add page, click the small add button on the top left of the page under the change password link.
Groups Add Page: To start, type the name of the group you wish to create in the "NAME" textbox. Then, either use the scroll bar, or the search box to select the combination of permission for the the members of the group. After clicking on the set of permissions, click the right arrow in between the two boxes to move that set of permissions into the chosen permissions box. This can be reversed by clicking the item in the chosen permissions box and proceeding to click the left arrow. If you feel that it is convenient, you can click the "Choose All" button to add all permissions or the remove all to remove all permissions from the group. At the bottom you can click the corresponding buttons to save and make a new group, save your progress and continue editing, or save and exit back to the main page (the save button).
Change Group Page: This page is identical to the add group page except the user can also click the red delete button at the bottom left to delete the group.
Users Select Page: Use the search bar to find the user you wish to edit. Conversely you can also use the several filters provided to you under the FILTER section to the right. By clicking the check box to the right of their name, you can then use the actions drop down menu to delete the ones you have selected. By clicking on a user’s name you get redirected to the change user page. To get to the user add page click the small add button on the top left of the page under the change password link.
Change User Page: The first textbox is used to change the username, under the personal info tab the user can fill out the first name, last name, and email address if he or she so desires. Under the permissions tab there are three check boxes which are active, staff status, and superuser status. Their effects are described for the user on the page. Under these you can select which groups you wish the user to belong to and click the right arrow to move them to the set of chosen groups. Below this the user will see set of 2 boxes for user permissions (see description of the add group page for further detail of how to use these boxes). Finally, the last section is for important dates such as last login and date joined, which can also be changed. At the bottom you can click the corresponding buttons to delete the user, save and make a new user, save your progress and continue editing, or save and exit back to the main page (the save button).
Users Add Page: To start, type the name of the user you wish to create in the NAME textbox. Then, enter the password for the new user and please retype your password into the password confirmation box to ensure you typed the correct password. At the bottom you can click the corresponding buttons to save and make a new user, save your progress and continue editing, or save and exit back to the main page (the save button).
Chemotherapy Drug Select Page: This page will display all the drugs in the database. By clicking the check box to the right of their name, you can then use the actions drop down menu to delete the ones you have selected. By clicking on one of the drug names you get redirected to the "Change Chemotherapy Drug" page. To get to the chemotherapy drug add page click the small add button on the top left of the page under the change password link.
Chemotherapy Drug Add Page: To start, type the name of the drug you wish to create in the NAME textbox. Then, enter the earliest and latest times the drug can be scheduled for any given day (you can click the now button to automatically enter the current time). After this is filled out, please enter any additional rules or pertinent information about the drug into the “Other Rules” box to be displayed when that drug is selected at the calendar view page. At the bottom you can click the corresponding buttons to save and make a new user, save your progress and continue editing, or save and exit back to the main page (the save button).
Change Chemotherapy Drug Page: This page is identical to the "Chemotherapy Add" page except one can delete the drug by clicking the red delete button and one can view the history of the drug relative to the database by clicking the grey "History" button at the top right under the "Log Out" link.
RN Schedule Select Page: This page will display all the RN schedules in the database. After clicking the check box to the right of their name, you can then use the actions drop down menu to delete the ones you have selected. By clicking on one of the RN schedule names you get redirected to the "Change RN Schedule" page.
Change RN Schedule Page: Fill out the name box to change the name of the saved schedule. Below this you can click the corresponding buttons to delete this schedule, save your progress and continue editing, or save and exit back to the main page (the save button).
Time Slot Inputs Select Page: This page will display all the sets of time slot inputs in the database. After clicking the check box to the right of their name, you can then use the actions drop down menu to delete the ones you have selected. By clicking on one of the time slot input names you get redirected to the change time slot inputs page.
Change Time Slot Inputs Page: Fill out the name box to change the name of the saved set of time slot inputs. Below this you can click the corresponding buttons to delete the selected set of time slot inputs, save your progress and continue editing, or save and exit back to the main page (the save button).
Logout Page: This page simply informs the user that he/she has been successfully logged out. By clicking the blue link under the text called "Log In Again" the user will be brought back to the login page for the application.
Change Password Page: As suggested by the name this page is for changing the user’s old password to a new one. Simply follow the instructions provided on this page by typing in the provided text boxes and click the CHANGE MY PASSWORD button on the middle right side of the page to change your password.

Technologies Explanation:

This web app was developed using the Pycharm IDE as it already had an excellent interface for building web tools using Django and Python. We chose to use Django with a Python backend because we wished to use new technologies to enhance our skillset and because we felt that Python is very easy to work with. As previously mentioned Python was used for the backend development which includes (but is not limited to) the algorithm for generating a schedule, communicating with the database, customizing the admin site, and grabbing information from the textboxes on the actual website.

For the font end we used the classic combination of HTML5 and CSS (enhanced with Bootstrap libraries) to design the web pages and make them aesthetically pleasing. We paired these with Javascript to add functionality to the various links and buttons that were placed on the website. Many of our Javascript functions utilized Ajax to call the various database related functions we had written in Python to either save, delete, or query for data (using Postgressql on Heroku , SQlite on local host). Ajax was chosen because it makes the page more responsive as it can make a call to the python script without causing a post back of the entire page. We used it solely for calls that interact with the database.

The website itself is (at the time of this writing) hosted on Heroku, which is a cloud server and is a temporary place to use the website until the client is ready to move the web app to one of their own servers. Heroku was chosen because it was very compatible to use with git which was our preferred version control software. To interact with git our team chose to use SourceTree as it is very eay to work with and has a great GUI (in our humble opinions).

Github: Version control software that allows for simultaneous editing
